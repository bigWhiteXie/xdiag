## 1. 概述

### 1.1 测试目的

本文档用于指导对 xdiag 智能诊断 CLI 工具的命令行功能进行全面测试，确保各命令功能的正确性、稳定性和用户体验。

### 1.2 测试范围

涵盖以下命令模块：
- **config**: 配置管理（model、set、show、unset、test）
- **target**: 目标资产管理（add、list、get、update、delete、test）
- **playbook**: 诊断剧本管理（list、show、update）
- **diag**: 智能诊断执行

### 1.3 命令结构

```
xdiag
├── config                    # 配置管理
│   ├── model                 # 配置 LLM 模型参数
│   ├── set <key> <value>     # 设置配置项
│   ├── show                  # 显示当前配置
│   ├── unset <key>           # 删除配置项
│   └── test                  # 测试 LLM 配置
├── target                    # 目标资产管理
│   ├── add                   # 添加目标
│   ├── list                  # 列出目标
│   ├── get                   # 获取目标详情
│   ├── update                # 更新目标
│   ├── delete                # 删除目标
│   └── test                  # 测试目标连通性
├── playbook                  # Playbook 管理
│   ├── list                  # 列出所有 Playbook
│   ├── show [name]           # 显示 Playbook 详情
│   └── update                # 更新 Playbook
└── diag                      # 执行智能诊断
```

## 2. 测试环境准备

### 2.1 前置条件

#### 系统要求
- macOS / Linux / Windows (with WSL)
- Go 1.21+ (仅开发/编译需要)
- Git (用于 playbook 更新)
- SQLite3 (自动集成，无需单独安装)

#### 依赖服务
- LLM API 访问权限（OpenAI / Anthropic / 自定义）
- 网络连接（用于 API 调用和 playbook 更新）

### 2.2 编译与安装

```bash
# 开发模式运行
cd /Volumes/upan/projects/xdiag
go run main.go --help

# 编译为可执行文件
go build -o xdiag

# 安装到系统路径
go install
```

### 2.3 测试数据清理

每次完整测试前，建议清理测试数据：

```bash
# 备份并清理配置目录
mv ~/.xdiag ~/.xdiag.backup
mkdir -p ~/.xdiag

# 验证环境干净
xdiag config show
# 应显示"未设置"或空配置
```

## 3. 配置命令测试

### 3.1 config model - 配置 LLM 模型参数

#### 测试用例 3.1.1: 配置 OpenAI 模型（基本场景）

**命令：**
```bash
xdiag config model \
  --api-key sk-your-api-key-here \
  --model-name gpt-4o
```

**预期输出：**
```
✅ 配置已保存到 ~/.xdiag/config.yaml
```

**验证步骤：**
```bash
xdiag config show
```

**检查点：**
- [ ] 配置文件成功创建在 `~/.xdiag/config.yaml`
- [ ] API Key 正确保存
- [ ] Model Name 正确保存
- [ ] Base URL 默认为 `https://api.openai.com/v1`
- [ ] Provider 默认为 `openai`

#### 测试用例 3.1.2: 配置自定义 LLM 服务

**命令：**
```bash
xdiag config model \
  --api-key custom-key-123 \
  --base-url https://custom.ai.com/v1 \
  --protocol openai \
  --model-name custom-model
```

**预期输出：**
```
✅ 配置已保存到 ~/.xdiag/config.yaml
```

**验证：**
```bash
xdiag config show
# 应显示自定义的 Base URL 和 Provider
```

#### 测试用例 3.1.3: 缺少必填参数

**命令：**
```bash
xdiag config model --model-name gpt-4o
```

**预期输出：**
```
Error: --api-key 是必填参数
```

**检查点：**
- [ ] 正确识别缺失参数
- [ ] 返回清晰的错误提示

#### 测试用例 3.1.4: 帮助信息

**命令：**
```bash

```

**预期输出：**
包含完整的参数说明和使用示例

### B. 自动化测试脚本示例

```bash
#!/bin/bash
# test_suite.sh - xdiag 自动化测试套件

set -e

echo "=== xdiag 自动化测试套件 ==="

# 清理环境
cleanup() {
    echo "清理测试环境..."
    rm -rf ~/.xdiag.test
    export XDIAG_CONFIG_DIR=~/.xdiag.test
}

# 测试配置命令
test_config() {
    echo "测试配置命令..."
    
    # 测试 model 配置
    xdiag config model --api-key test-key --model-name gpt-4o
    if [ $? -ne 0 ]; then
        echo "❌ config model 失败"
        exit 1
    fi
    
    # 测试 show
    xdiag config show | grep "gpt-4o"
    if [ $? -ne 0 ]; then
        echo "❌ config show 失败"
        exit 1
    fi
    
    echo "✅ 配置命令测试通过"
}

# 测试目标管理
test_targets() {
    echo "测试目标管理..."
    
    # 添加目标
    xdiag target add --name test-server --kind node --address 192.168.1.1 --port 22
    if [ $? -ne 0 ]; then
        echo "❌ target add 失败"
        exit 1
    fi
    
    # 列出目标
    xdiag target list | grep "test-server"
    if [ $? -ne 0 ]; then
        echo "❌ target list 失败"
        exit 1
    fi
    
    echo "✅ 目标管理测试通过"
}

# 主测试流程
main() {
    cleanup
    test_config
    test_targets
    echo ""
    echo "=== 所有测试通过 ✅ ==="
}

main
```

### D. 已知问题和限制

1. **Playbook 更新依赖 Git**: 必须预先安装 git 命令

4. **目标类型**: 目前仅支持 node/postgres/mysql/redis
