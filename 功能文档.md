# xdiag 智能诊断 CLI 工具 - 完整功能规格文档

## 一、项目概述

### 1.1 产品定位
xdiag 是一个基于 Go 语言和 Eino 框架开发的**非交互式、跨节点、Agent 驱动的智能诊断 CLI 工具**。它通过配置化的资产管理、剧本化的诊断方案、以及 AI Agent 的自主决策能力，实现自动化故障诊断和修复。

### 1.2 核心价值
- **配置驱动资产**：统一管理待诊断的目标节点（主机、数据库、中间件等）
- **剧本驱动能力**：将专家经验转化为可执行的诊断剧本（Playbook）
- **Agent 驱动逻辑**：利用大模型的推理能力，自主选择诊断路径并执行

### 1.3 技术栈
| 组件 | 技术选型 | 用途 |
|------|----------|------|
| CLI 框架 | spf13/cobra v1.10.2 | 命令行解析和命令组织 |
| 配置管理 | spf13/viper v1.21.0 | 配置文件解析、环境变量支持 |
| Agent 编排 | Eino (字节智能体框架) | 多阶段 Agent Pipeline 编排 |
| 远程执行 | golang.org/x/crypto/ssh | SSH 连接和命令执行 |
| 本地存储 | SQLite | 目标资产、诊断历史持久化 |
| 自动升级 | GitHub Releases API | 二进制文件热更新 |

**1. SQLite 数据表设计**

```sql
-- internal/store/migrations.go
CREATE TABLE IF NOT EXISTS targets (
    id TEXT PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    kind TEXT NOT NULL CHECK(kind IN ('host', 'mysql', 'redis', 'k8s_cluster')),
    ip TEXT NOT NULL,
    port INTEGER DEFAULT 22,
    username TEXT NOT NULL,
    password TEXT,  -- 暂时明文存储（后续版本加密）
    ssh_key_path TEXT,
    tags TEXT,  -- JSON 数组格式：["prod","db"]
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_checked DATETIME,
    status TEXT DEFAULT 'unknown' CHECK(status IN ('connected', 'failed', 'unknown'))
);

CREATE INDEX idx_targets_tags ON targets(tags);
CREATE INDEX idx_targets_kind ON targets(kind);
```

**2. Target 数据模型**
```go
// internal/target/model.go
type Target struct {
    ID          string   `json:"id"`
    Name        string   `json:"name"`
    Kind        string   `json:"kind"` // host, mysql, redis, k8s_cluster
    IP          string   `json:"ip"`
    Port        int      `json:"port"`
    Username    string   `json:"username"`
    Password    string   `json:"password,omitempty"`
    SSHKeyPath  string   `json:"ssh_key_path,omitempty"`
    Tags        []string `json:"tags"`
    CreatedAt   time.Time `json:"created_at"`
    UpdatedAt   time.Time `json:"updated_at"`
    LastChecked *time.Time `json:"last_checked,omitempty"`
    Status      string   `json:"status"` // connected, failed, unknown
}

// 生成唯一 ID
func GenerateTargetID(kind, ip string) string {
    hash := md5.Sum([]byte(fmt.Sprintf("%s-%s", kind, ip)))
    return fmt.Sprintf("target-%x", hash[:8])
}
```

**SSH 连接检测（Host 类型）**
```go
// internal/target/connectivity.go
func CheckSSHConnection(target *Target) error {
    config := &ssh.ClientConfig{
        User: target.Username,
        Auth: []ssh.AuthMethod{},
        HostKeyCallback: ssh.InsecureIgnoreHostKey(),
        Timeout: 3 * time.Second,
    }
    
    if target.Password != "" {
        config.Auth = append(config.Auth, ssh.Password(target.Password))
    } else if target.SSHKeyPath != "" {
        key, err := os.ReadFile(target.SSHKeyPath)
        if err != nil {
            return fmt.Errorf("读取 SSH Key 失败：%v", err)
        }
        signer, err := ssh.ParsePrivateKey(key)
        if err != nil {
            return fmt.Errorf("解析 SSH Key 失败：%v", err)
        }
        config.Auth = append(config.Auth, ssh.PublicKeys(signer))
    }
    
    addr := fmt.Sprintf("%s:%d", target.IP, target.Port)
    client, err := ssh.Dial("tcp", addr, config)
    if err != nil {
        return fmt.Errorf("SSH 连接失败：%v", err)
    }
    defer client.Close()
    
    // 执行简单命令验证连接
    session, err := client.NewSession()
    if err != nil {
        return err
    }
    defer session.Close()
    
    err = session.Run("echo 'connection_test'")
    return err
}
```

**4. Target 仓储操作**
```go
// internal/store/target_repo.go
type TargetRepo struct {
    db *sql.DB
}

func (r *TargetRepo) Add(ctx context.Context, target *Target) error {
    // 生成 ID
    target.ID = GenerateTargetID(target.Kind, target.IP)
    
    // 插入或更新
    query := `
        INSERT INTO targets (id, name, kind, ip, port, username, password, ssh_key_path, tags)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
        ON CONFLICT(id) DO UPDATE SET
            name = excluded.name,
            kind = excluded.kind,
            ip = excluded.ip,
            port = excluded.port,
            username = excluded.username,
            password = excluded.password,
            ssh_key_path = excluded.ssh_key_path,
            tags = excluded.tags,
            updated_at = CURRENT_TIMESTAMP
    `
    
    tagsJSON, _ := json.Marshal(target.Tags)
    _, err := r.db.ExecContext(ctx, query,
        target.ID, target.Name, target.Kind, target.IP, target.Port,
        target.Username, target.Password, target.SSHKeyPath, string(tagsJSON))
    
    return err
}

func (r *TargetRepo) ListByTags(ctx context.Context, tags []string) ([]*Target, error) {
    // 模糊匹配标签（SQLite JSON 查询）
    query := `SELECT * FROM targets WHERE tags LIKE ?`
    // 简化实现：使用字符串匹配（生产环境建议用 JSON 函数）
    tagPattern := "%" + strings.Join(tags, "%") + "%"
    
    rows, err := r.db.QueryContext(ctx, query, tagPattern)
    if err != nil {
        return nil, err
    }
    defer rows.Close()
    
    var targets []*Target
    for rows.Next() {
        var t Target
        var tagsStr string
        scanErr := rows.Scan(&t.ID, &t.Name, &t.Kind, &t.IP, &t.Port,
            &t.Username, &t.Password, &t.SSHKeyPath, &tagsStr,
            &t.CreatedAt, &t.UpdatedAt, &t.LastChecked, &t.Status)
        if scanErr != nil {
            return nil, scanErr
        }
        json.Unmarshal([]byte(tagsStr), &t.Tags)
        targets = append(targets, &t)
    }
    return targets, nil
}
```

### 2.3 Playbook 管理系统

#### 功能描述
Playbook 是诊断方案的核心载体，采用分层渐进式结构。每个 Playbook 包含元数据（metadata.yaml）和多个诊断步骤（refs/*.yaml）。Agent 根据问题描述和目标的标签，自动选择合适的 Playbook 并执行诊断。

#### Playbook 目录结构
```
~/.xdiag/playbooks/
├── xos-deploy/                 # 一级分类：xos 部署方向
│   ├── metadata.yaml           # 元数据
│   └── refs/
│       ├── disk_full.yaml      # 二级分类：磁盘满诊断
│       ├── slow_network.yaml   # 二级分类：网络慢诊断
│       └── startup_fail.yaml   # 二级分类：启动失败诊断
│
├── k8s-troubleshoot/           # 一级分类：K8s 问题诊断
│   ├── metadata.yaml
│   └── refs/
│       ├── pod_crashloop.yaml
│       ├── service_unreachable.yaml
│       └── dns_resolution_fail.yaml
│
└── mysql-performance/          # 一级分类：MySQL 性能优化
    ├── metadata.yaml
    └── refs/
        ├── slow_query.yaml
        ├── lock_contention.yaml
```

```go
// internal/playbook/loader.go
type PlaybookLoader struct {
    basePath string
}

type Playbook struct {
    Metadata      *Metadata  `yaml:"metadata"`
    DiagnosisRefs []*StepRef `yaml:"refs"`
}

type Metadata struct {
    Name           string   `yaml:"name"`
    DisplayName    string   `yaml:"display_name"`
    Description    string   `yaml:"description"`
    Version        string   `yaml:"version"`
    ApplicableKinds []string `yaml:"applicable_kinds"`
    RequiredTags   []string `yaml:"required_tags"`
    OptionalTags   []string `yaml:"optional_tags"`
    DiagnosisPaths []DiagnosisPath `yaml:"diagnosis_paths"`
}

type DiagnosisPath struct {
    ID          string   `yaml:"id"`
    Name        string   `yaml:"name"`
    Description string   `yaml:"description"`
    Refs        []string `yaml:"refs"`
}

type StepRef struct {
    Name     string `yaml:"name"`
    Content  *Step  `yaml:"content"`
}

type Step struct {
    ID             string            `yaml:"id"`
    Name           string            `yaml:"name"`
    Command        string            `yaml:"command"`
    Expect         Expectation       `yaml:"expect"`
    OnMatch        *ActionResult     `yaml:"on_match"`
    OnNoMatch      *ActionResult     `yaml:"on_no_match"`
    OnSuccess      *ActionResult     `yaml:"on_success"`
    OnFailure      *ActionResult     `yaml:"on_failure"`
}

type Expectation struct {
    OutputPattern string `yaml:"output_pattern"`
    Condition     string `yaml:"condition"`  // 表达式，用 eval 解析
    Action        string `yaml:"action"`     // record, analyze, remediate
}

type ActionResult struct {
    NextStep   string `yaml:"next_step"`
    Conclusion string `yaml:"conclusion"`
    Issue      string `yaml:"issue"`
    Exit       bool   `yaml:"exit"`
}

func (l *PlaybookLoader) Load(playbookName string) (*Playbook, error) {
    playbookPath := filepath.Join(l.basePath, playbookName)
    
    // 加载 metadata.yaml
    metadataFile := filepath.Join(playbookPath, "metadata.yaml")
    metadataBytes, err := os.ReadFile(metadataFile)
    if err != nil {
        return nil, fmt.Errorf("加载元数据失败：%v", err)
    }
    
    var metadata Metadata
    yaml.Unmarshal(metadataBytes, &metadata)
    
    // 加载所有 refs/*.yaml
    refsDir := filepath.Join(playbookPath, "refs")
    files, err := os.ReadDir(refsDir)
    if err != nil {
        return nil, err
    }
    
    var refs []*StepRef
    for _, file := range files {
        if !strings.HasSuffix(file.Name(), ".yaml") {
            continue
        }
        
        content, err := os.ReadFile(filepath.Join(refsDir, file.Name()))
        if err != nil {
            return nil, err
        }
        
        var step Step
        yaml.Unmarshal(content, &step)
        
        refs = append(refs, &StepRef{
            Name:    file.Name(),
            Content: &step,
        })
    }
    
    return &Playbook{
        Metadata: &metadata,
        DiagnosisRefs: refs,
    }, nil
}
```

```go
func (r *PlaybookRegistry) FindByTags(kind string, tags []string) []*Playbook {
    var matched []*Playbook
    
    for _, pb := range r.playbooks {
        // 检查适用类型
        if !contains(pb.Metadata.ApplicableKinds, kind) {
            continue
        }
        
        // 检查必需标签
        hasRequired := true
        for _, reqTag := range pb.Metadata.RequiredTags {
            if !contains(tags, reqTag) {
                hasRequired = false
                break
            }
        }
        
        if hasRequired {
            matched = append(matched, pb)
        }
    }
    
    return matched
}

// 获取所有诊断方向（用于 Agent 选择）
func (r *PlaybookRegistry) GetDiagnosisDirections(playbookName string) []DiagnosisPath {
    pb := r.playbooks[playbookName]

        return nil
    }
    return pb.Metadata.DiagnosisPaths
```

```go
// internal/agent/router.go
func (p *DiagnosticPipeline) RouteTarget(ctx context.Context, query string) (*RouteTargetResult, error) {
    // 构建 Prompt，让 LLM 从用户描述中提取目标信息
    prompt := fmt.Sprintf(`
你是一个智能诊断助手。请分析用户的故障描述，从中提取诊断目标的信息。

用户描述：%s

可用目标列表：
`, query)

    // 从数据库加载所有目标（带标签）
    targets, err := p.targetRepo.ListAll(ctx)
    if err != nil {
        return nil, err
    }

    for _, t := range targets {
        prompt += fmt.Sprintf("- ID: %s, Name: %s, Kind: %s, IP: %s, Tags: %v\n",
            t.ID, t.Name, t.Kind, t.IP, t.Tags)
    }

    prompt += `
请根据用户描述，匹配最相关的目标。输出 JSON 格式：
{
    "kind": "目标类型 (host/mysql/...)",
    "names": ["目标名称列表"]
}

如果用户明确提到了 IP 或名称，优先匹配；否则根据标签和上下文推断。
`

    // 调用 LLM（使用 Function Call）
    resp, err := p.llmClient.Generate(ctx, prompt,
        eino.WithResponseFormat(eino.ResponseFormatJSON),
        eino.WithTools(p.createTargetQueryTool()),
    )
    if err != nil {
        return nil, err
    }

    var result RouteTargetResult
    json.Unmarshal([]byte(resp.Content), &result)
    return &result, nil
}

// 创建目标查询工具（供 LLM 调用）
func (p *DiagnosticPipeline) createTargetQueryTool() *eino.Tool {
    return &eino.Tool{
        Name:        "query_targets",
        Description: "查询目标数据库，根据 IP、名称、标签筛选目标",
        Handler: func(args map[string]interface{}) (interface{}, error) {
            // LLM 可以调用此工具来搜索目标
            filter, _ := args["filter"].(string)
            targets, err := p.targetRepo.Search(context.Background(), filter)
            return targets, err
        },
    }
}
```

**4. 阶段三：Executor Agent（ReAct 模式）**

```go
// internal/agent/executor.go
type ExecutorAgent struct {
    llmClient    *eino.Client
    sshTool      *tools.SSHTool
    playbookSteps []*playbook.Step
    maxIterations int
}

func (e *ExecutorAgent) Execute(ctx context.Context, target *Target, playbook *playbook.Playbook) (*ExecutionResult, error) {
    result := &ExecutionResult{
        PlaybookName: playbook.Metadata.Name,
        StepsExecuted: make([]StepExecution, 0),
        IssuesFound: make([]Issue, 0),
    }

    // 遍历所有诊断步骤
    for _, stepRef := range playbook.DiagnosisRefs {
        step := stepRef.Content
        
        // 为每个 step 创建一个 ReAct Agent
        agentPrompt := fmt.Sprintf(`
你是一个专业的系统运维专家。现在需要诊断目标节点的问题。

目标节点：%s (%s)
当前诊断步骤：%s
需要执行的命令：%s

请执行以下步骤：
1. 通过 SSH 在目标节点执行命令
2. 分析命令输出
3. 根据输出判断是否存在问题
4. 记录你的发现

命令输出：
`, target.Name, target.IP, step.Name, step.Command)

        // 创建 SSH 执行工具
        sshTool := e.sshTool.WithTarget(target)
        
        // 执行命令
        output, err := sshTool.RunCommand(step.Command)
        if err != nil {
            // 命令执行失败也是一种发现
            result.IssuesFound = append(result.IssuesFound, Issue{
                Severity:    "critical",
                Description: fmt.Sprintf("命令执行失败：%s", step.Name),
                Evidence:    err.Error(),
            })
            continue
        }

        agentPrompt += output

        // 让 LLM 分析输出
        resp, err := p.llmClient.Generate(ctx, agentPrompt)
        if err != nil {
            continue
        }

        // 解析 LLM 的分析结果
        stepExec := StepExecution{
            StepID:     step.ID,
            Command:    step.Command,
            Output:     output,
            Conclusion: resp.Content,
        }

        // 检查是否匹配预期模式
        if step.Expect.OutputPattern != "" {
            re := regexp.MustCompile(step.Expect.OutputPattern)
            matches := re.FindStringSubmatch(output)
            stepExec.Matched = len(matches) > 0
            
            if stepExec.Matched && step.Expect.Condition != "" {
                // 评估条件表达式（简单的变量替换）
                conditionMet := e.evaluateCondition(step.Expect.Condition, matches)
                if !conditionMet {
                    stepExec.Matched = false
                }
            }
        }

        result.StepsExecuted = append(result.StepsExecuted, stepExec)

        // 如果步骤定义了退出条件，提前结束
        if step.OnMatch != nil && step.OnMatch.Exit && stepExec.Matched {
            result.Conclusion = step.OnMatch.Conclusion
            break
        }
        if step.OnNoMatch != nil && step.OnNoMatch.Exit && !stepExec.Matched {
            result.Conclusion = step.OnNoMatch.Conclusion
            break
        }
    }

    return result, nil
}

// 评估条件表达式（简化版本，实际可用 expr 库）
func (e *ExecutorAgent) evaluateCondition(condition string, matches []string) bool {
    // 示例： "match.group(2) > 80"
    // 替换 match.group(N) 为实际值
    for i, m := range matches {
        placeholder := fmt.Sprintf("match.group(%d)", i)
        condition = strings.ReplaceAll(condition, placeholder, m)
    }
    
    // 简单的布尔表达式求值（生产环境用 expr 或 eval）
    // 这里只是示意，实际需要安全的表达式解析
    return strings.Contains(condition, "true")
}
```

```go
// internal/agent/executor.go
func (p *DiagnosticPipeline) ExecuteMultiple(
    ctx context.Context,
    target *Target,
    selections []*PlaybookSelection,
) ([]*ExecutionResult, error) {
    
    // 使用 goroutine 并发执行
    results := make([]*ExecutionResult, len(selections))
    errors := make([]error, len(selections))
    
    var wg sync.WaitGroup
    for i, selection := range selections {
        wg.Add(1)
        go func(idx int, sel *PlaybookSelection) {
            defer wg.Done()
            
            playbook := p.playbookRegistry.Get(sel.PlaybookName)
            executor := p.createExecutorForPlaybook(playbook)
            
            results[idx], errors[idx] = executor.Execute(ctx, target, playbook)
        }(i, selection)
    }
    
    wg.Wait()
    
    // 收集错误
    var errs []error
    for _, err := range errors {
        if err != nil {
            errs = append(errs, err)
        }
    }
    
    return results, multierror.Append(nil, errs...)
}
```

```go
    if len(issues) == 0 {

    for _, issue := range issues {
        emoji := "🔴"
        if issue.Severity == "warning" {
            emoji = "🟡"
        } else if issue.Severity == "info" {
            emoji = "ℹ️"
        }
        g.buffer.WriteString(fmt.Sprintf("   %s [%s] %s\n", emoji, issue.Severity, issue.Description))
        g.buffer.WriteString(fmt.Sprintf("      证据：%s\n", issue.Evidence))
    }

            manualFixes = append(manualFixes, rec)

    if len(manualFixes) > 0 {
        g.buffer.WriteString("   【手动修复】\n")
        for i, rec := range manualFixes {
            g.buffer.WriteString(fmt.Sprintf("   %d. %s\n", i+1, rec.Title))
            if rec.Command != "" {
                g.buffer.WriteString(fmt.Sprintf("      %s\n", rec.Command))
            }
            g.buffer.WriteString(fmt.Sprintf("      %s\n", rec.Description))
        }
    }
    
    g.buffer.WriteString("\n")
}

    g.buffer.WriteString(fmt.Sprintf("   - 执行命令数：%d\n", stats.CommandsExecuted))
    g.buffer.WriteString(fmt.Sprintf("   - 发现问题数：%d\n", stats.IssuesFound))
    g.buffer.WriteString(fmt.Sprintf("   - 总耗时：%s\n", stats.Duration.Round(time.Second)))
    g.buffer.WriteString(fmt.Sprintf("   - 使用的 Playbook：%s\n", strings.Join(stats.PlaybooksUsed, ", ")))
    g.buffer.WriteString("\n==================================================\n")
```

```go
// internal/updater/binary_upgrader.go
type BinaryUpgrader struct {
    githubRepo string  // "your-org/xdiag"
    currentBin string  // 当前二进制路径
}

type Release struct {
    TagName    string `json:"tag_name"`
    Name       string `json:"name"`
    Assets     []ReleaseAsset `json:"assets"`
}

type ReleaseAsset struct {
    Name   string `json:"name"`
    BrowserDownloadURL string `json:"browser_download_url"`
}

func (u *BinaryUpgrader) Upgrade(targetVersion string) error {
    // 1. 获取最新版本
    releases, err := u.fetchReleases()
    if err != nil {
        return fmt.Errorf("获取版本信息失败：%v", err)
    }
    
    var targetRelease *Release
    if targetVersion == "" {
        // 使用最新版本
        targetRelease = &releases[0]
    } else {
        // 查找指定版本
        for _, r := range releases {
            if r.TagName == targetVersion {
                targetRelease = &r
                break
            }
        }
        if targetRelease == nil {
            return fmt.Errorf("未找到版本 %s", targetVersion)
        }
    }
    
    // 2. 选择对应平台的二进制
    asset := u.selectAsset_for_platform(targetRelease.Assets)
    if asset == nil {
        return fmt.Errorf("不支持当前平台")
    }
    
    // 3. 下载到临时目录
    tmpDir := "/tmp/xdiag-upgrade"
    os.MkdirAll(tmpDir, 0755)
    tmpBin := filepath.Join(tmpDir, "xdiag-new")
    
    resp, err := http.Get(asset.BrowserDownloadURL)
    if err != nil {
        return err
    }
    defer resp.Body.Close()
    
    out, err := os.Create(tmpBin)
    if err != nil {
        return err
    }
    _, err = io.Copy(out, resp.Body)
    if err != nil {
        return err
    }
    out.Close()
    
    // 4. 赋予执行权限
    os.Chmod(tmpBin, 0755)
    
    // 5. 替换当前二进制（关键步骤）
    // 使用 self-replacement 技巧：创建子进程替换父进程的文件
    err = u.replace_self(tmpBin)
    if err != nil {
        return fmt.Errorf("替换失败：%v", err)
    }
    
    return nil
}

func (u *BinaryUpgrader) replace_self(newBin string) error {
    // 获取当前进程路径
    currentBin, err := os.Executable()
    if err != nil {
        return err
    }
    
    // 重命名旧文件（备份）
    backupBin := currentBin + ".old"
    err = os.Rename(currentBin, backupBin)
    if err != nil {
        return err
    }
    
    // 移动新文件到原位置
    err = os.Rename(newBin, currentBin)
    if err != nil {
        // 回滚
        os.Rename(backupBin, currentBin)
        return err
    }
    
    // 删除备份
    os.Remove(backupBin)
    
    return nil
}

func (u *BinaryUpgrader) select_asset_for_platform(assets []ReleaseAsset) *ReleaseAsset {
    // 检测当前平台
    goos := runtime.GOOS  // linux, darwin, windows
    goarch := runtime.GOARCH  // amd64, arm64
    
    // 匹配规则：xdiag-{os}-{arch}
    pattern := fmt.Sprintf("xdiag-%s-%s", goos, goarch)
    
    for _, asset := range assets {
        if strings.Contains(asset.Name, pattern) {
            return &asset
        }
    }
    
    return nil
}
```

```go
func (u *PlaybookUpdater) Update() error {
    // 1. 克隆或更新 GitHub 仓库到临时目录
    tmpDir := "/tmp/xdiag-playbooks"
    
    if _, err := os.Stat(tmpDir); os.IsNotExist(err) {
        // 首次克隆
        cmd := exec.Command("git", "clone", 
            fmt.Sprintf("https://github.com/%s.git", u.githubRepo),
            tmpDir)
        err := cmd.Run()
        if err != nil {
            return fmt.Errorf("克隆仓库失败：%v", err)
        }
    } else {
        // 已有仓库，执行 pull
        cmd := exec.Command("git", "-C", tmpDir, "pull")
        err := cmd.Run()
        if err != nil {
            return fmt.Errorf("更新仓库失败：%v", err)
        }
    }
    
    // 2. 对比文件变化
    changedFiles, err := u.detect_changes(tmpDir, u.playbooksDir)
    if err != nil {
        return err
    }
    
    // 3. 覆盖更新
    for _, file := range changedFiles {
        src := filepath.Join(tmpDir, file)
        dst := filepath.Join(u.playbooksDir, file)
        
        // 确保目录存在
        os.MkdirAll(filepath.Dir(dst), 0755)
        
        // 复制文件
        srcData, err := os.ReadFile(src)
        if err != nil {
            return err
        }
        err = os.WriteFile(dst, srcData, 0644)
        if err != nil {
            return err
        }
        
        fmt.Printf("📥 更新：%s\n", file)
    }
    
    return nil
}
```

- **多 Playbook 并行**：对同一目标并行执行多个诊断方案
- **结果聚合**：自动聚合多个诊断结果，去重和优先级排序

### 4.3 安全机制
- **凭证管理**：未来版本支持 AES 加密存储密码
- **SSH Key 优先**：推荐使用 SSH 免密登录
- **审计日志**：记录所有执行的命令和操作

### 4.4 扩展性
- **自定义工具**：支持为 Agent 绑定自定义工具（HTTP API、数据库查询等）
- **插件系统**：未来支持第三方插件扩展 Playbook 执行器
- **Webhook 通知**：诊断完成后可触发 Webhook

## 五、最佳实践

### 5.1 目标管理最佳实践
1. **统一命名规范**：使用有意义的名称（prod-db-01, staging-web-02）
2. **标签标准化**：制定团队统一的标签体系
3. **定期健康检查**：使用 `xdiag target test` 定期检查连通性
4. **分组管理**：通过标签区分环境、角色、重要性

### 5.2 Playbook 编写最佳实践
1. **原子化步骤**：每个 step 只做一件事
2. **明确的退出条件**：定义清晰的成功/失败标准
3. **幂等性**：修复操作应该是幂等的
4. **充分的注释**：为每个步骤添加详细描述
5. **版本控制**：Playbook 应该纳入 Git 版本管理

### 5.3 诊断流程最佳实践
1. **精确描述问题**：提供详细的症状、时间、影响范围
2. **先观察后修复**：先诊断清楚再执行修复
3. **保留现场**：重要操作前先备份配置
4. **验证修复效果**：修复后再次验证问题是否解决
